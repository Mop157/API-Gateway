запрос на инструмент вручную:

{
    "ip": "название домена или айпи", // строка
    "setting": {
      "list_port": "диапазон портов", // строка
      "arguments": ["аргументы при сканиворание"] // массив с аргументами
    }
}

пример:

{
    "ip": "google.com",
    "setting": {
      "list_port": "40,80",
      "arguments": ["-A", "-sV"]
    }
}

описание:
вручная настройка для сканиворание с помошю инструмента Nmap

--------------------------------------------------------------------------------------

запрос на инструмент готовые варианты:

{
    "ip": "название домена или айпи", // строка
    "script": "название скрипта" // строка
}

пример:

{
    "ip": "google.com",
    "script": "Firewall Evasion"
}

описание:
автоматический скан сети с помощю скрипта

доступные скрипты:

"fyll server scan" - Полное сканирование сервера с обнаружением версий и уязвимостей
Цель: Максимально полное исследование хоста на уязвимости и открытые порты. (от 10м до 1.30ч)

"NSE-scripts" - Сканирование с использованием интенсивных NSE-скриптов
Цель: Проверка на уязвимости с помощью агрессивных скриптов (только на тестовых серверах!). (от 10м до 1.30ч)

"TCP and UDP scan" - Глубокое TCP и UDP сканирование с определением сервисов
Цель: Комплексное сканирование сети на TCP/UDP-уровнях с идентификацией сервисов и ОС.

"web-scan" - Быстрое сканирование на уязвимые веб-серверы
Цель: Оценка безопасности веб-серверов, поиск стандартных уязвимостей и конфигурационных проблем.

"Firewall Evasion" - Сканирование с обходом сетевых фильтров
Цель: Сканирование, скрывающее исходный IP для обхода фаерволов или IDS.

"vulnerable SMB services" - Поиск уязвимых сервисов SMB (например, EternalBlue)
Цель: Идентификация уязвимых SMB-серверов.

"Security Audit for DNS Servers" - Аудит безопасности для DNS-серверов
Цель: Оценка безопасности DNS-конфигурации и предотвращение утечек информации.

--------------------------------------------------------------------------------------------------------

авторизация: обязательно
выбор языка: не обязательно ( по стандарту EN )

--------------------------------------------------------------------------------------------------------

неправильные запросы:

{
    "ip": "название домена или айпи", // строка
    "setting": {
      "list_port": "диапазон портов", // строка
      "arguments": ["аргументы при сканиворание"] // массив с аргументами
    },
    "script": "название скрипта" // строка
}

описание:
в данном запросе используеться как вручная настройка так и скрипт что являеться неправильним так как можно использовать только одно, в данном примере приоритет будет за скриптом
